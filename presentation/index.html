Javascript: The Basics

You all know Javascript, right?

History

Javascript actually was...

  • Created at Netscape by Brendan Eich.
  • Supposed to be a compliment to Java.
  • Originally called LiveScript.
  • Renamed to JavaScript to help Java compete with C++.
  • Not directly based on Java, despite the name.
  • Submitted as a standard in 1994, renamed to ECMAScript.

So, there's only one version...

  • No, there's at least five major variants.
    • ECMAScript - Original standard (used in Chrome, ChromeOS, Opera, node.js)
    • JavaScript - Mozilla's implementation (used in Firefox, Thunderbird, Firefox OS)
    • JScript - Microsoft's implementation (used in IE, Windows 8, and various MS Products)
    • ActionScript - Adobe's implementation (used as scripting for Flash)
    • JSHarmony - Javascript: The Next Generation (proposal for ECMAScript)
  • And each has multiple versions!

...what are we talking about then?

  • We're talking about ECMAScript 5.
  • Why?
    • It's the most common subset.

Functions

That's no moon!

There are several ways to write a function.

  • function foo( ) {...}
  • var foo = function( ) {...}

Which is right?

  • It depends.

Let's Explain...

Oh, by the way, functions are also objects.

...Seriously?!

Functions can also have properties.

function foo()
{
// Some code here.
};

foo.bar = "Hi. I'm a property.";

// Prints "Hi. I'm a property."
console.log(foo.bar);

Wait, what?

But wait, there's more!

Where are my classes?

  • Javascript has no concept of classes.
    • This is totally different from Java, C++, C#, or basically any other major object-oriented language!
    • Trying to use Javascript like Java will only lead to pain!
  • Instead, Javascript has Prototypes.

Lemme s'plain...

In Javascript, the only thing you have is objects.

  • A function is used as a Constructor for object instances.
  • Instead of a class, the Constructor has an object called a Prototype that contains fallbacks for any instance's properties.
  • The Prototype is set as a property named 'prototype' on the Constructor.
  • Any object can be a Prototype.
  • Instances can override properties from the Prototype.
  • The Prototype is shared by all instances of a Constructor.
  • Instances must be created with new in order to inherit from their Constructor's Prototype.

Remember, failure is always an option!

Scope

Minty Fresh

  • When we use the term "scope", we mean the parts of the code that can access whatever we're talking about.
  • In Javascript, a variable's scope starts at the beginning of where it's defined, and includes everything "below" it.
  • Only declaration location matter for scope, not where something is called from. 
    function foo(callback) {
    // Prints "yo."
    callback();
}

function bar() {
    var test = "yo.";
    function cb() { console.log(test); }
    foo(cb);
}

Honorable Mentions

There are lots of tools out there that help.

  • jQuery - A lightweight framework designed to ease use of javascript.
    • Used by 56.7% of all websites.
    • Used by 91.7% of all sites using external javascript libraries.
  • Prototype - provides a simple api for common web tasks.
    • Extends the existing Javascript objects.
    • Has a more conventional Class system.
  • MooTools - An object-oriented framework for writing cross-browser web applications.
    • Focuses on extending core Javascript language.
    • Attempts to hide differences between different implementations.

And thousands more!
